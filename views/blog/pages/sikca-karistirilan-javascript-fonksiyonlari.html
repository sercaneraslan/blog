Sıkça Karıştırılan JavaScript Fonksiyonları

10 Ağustos 2020

Birçok JavaScript geliştiricisi JavaScript’in bazı gömülü (built-in) fonksiyonlarını binlerce kez kullanmış olsa dahi zaman zaman bu fonksiyonları birbirleriyle karıştırabiliyor. Bu makalede isimleri ve ne iş yaptıklarını “ezberlemeden” alternatif yöntemlerle “Aklımızda nasıl tutarız?” sorusuna cevaplar arayacağız.

Öncelikle “İnsanlar nasıl öğrenir ve nasıl hatırlar?” bu konuya bir göz atalım.

İnsanlar öğrenme işini kıyas yöntemiyle yaparlar. Kıyas yöntemi önceden öğrenilen bilgiler ile yeni öğrenilen bilgileri ilişkilendirerek yapılır. Bu ilişkilendirme ne kadar güçlü olursa hatırlamak o kadar kolay olur. Dolayısıyla ilişkilendirmeyi güçlendirmeye odaklanmalıyız.

Başlamadan önce, bulacağınız cevapların deneysel olduğunu belirtmek isterim :)

map(), filter(), reduce()

Fonksiyonları hatırlayalım;

    
// Her bir elemanı değiştirir.
[1, 3, 5].map(sayi => sayi * 2); // [2, 6, 10]

// Tüm elemanları filtreler.
[1, 3, 5].filter(sayi => sayi > 3); // [5]

// Tüm elemanlardan yeni bir eleman üretir.
[1, 3, 5].reduce((toplam, sayi) => toplam + sayi); // 9

Aklımızda nasıl tutarız?

Fonksiyon isimlerine benzer kelimeler bularak ne iş yaptıklarını hatırlamaya çalışalım.

  • map() fonksiyonunu hatırlamak için manipüle kelimesi bizim için harika bir seçenek. map() fonksiyonunu gördüğümüz anda manipüle kelimesini hatırlarsak map() fonksiyonunun elemanları manipüle ettiğini kolayca hatırlayabiliriz.
  • filter() fonksiyonunun elemanları filtrelediğini isminden dolayı zaten hemen hatırlıyoruz :)
  • reduce() fonksiyonun son 2 harfinden “Create Element” çıkarımını yaparsak map() fonksiyonundan farklı olarak yeni bir eleman ürettiğini hatırlayabiliriz.

map(), forEach()

Fonksiyonları hatırlayalım;

map()

  • Verilen array’den yeni bir array üretir.
  • Performans olarak daha hızlıdır.
  • Chaining yaparak reduce() ve filter() ile birleştirilebilir.

forEach()

  • Verilen array üzerinde çalışır.
  • JavaScript’in for ifadesine benzetilebilir.

Aklımızda nasıl tutarız?

  • forEach() fonksiyonunu gördüğümüzde aklımıza for kelimesini getirebiliriz. for ifadesinin basit bir şekilde çalıştığını bildiğimiz için yeni bir array üretmediğini var olan array üzerinde çalıştığını hatırlayabiliriz.
  • map() fonksiyonunun son harfini performans kelimesi ile eşleştirirsek forEach’e göre daha performanslı olduğunu hatırlayabiliriz. forEach() fonksiyonun yeni array üretmediğini hatırladığımızda map() fonksiyonunun yeni bir array ürettiğini hatırlayabiliriz.

ceil(), floor(), round()

Fonksiyonları hatırlayalım;

    
// Sayıyı yukarı yuvarlar.
Math.ceil(3.75); // 4

// Sayıyı aşağı yuvarlar.
Math.floor(3.75); // 3

// .50 ve üzerini yukarı .49 ve altını aşağı yuvarlar.
Math.round(3.50); // 4
Math.round(3.49); // 3

Aklımızda nasıl tutarız?

  • Alfabetik olarak fonksiyonları zihnimizde sıraladığımızda, alfabetik sırada en yukarıda olan ceil() fonksiyonun sayıyı yukarı yuvarladığını rahatça hatırlayabiliriz.
  • Yukarı yuvarlayan fonksiyonun hemen aşağısında yer alan floor() fonksiyonun da aşağı yuvarladığını rahatça hatırlayabiliriz.
  • round() fonksiyonu alfabetik sırada diğer 2 fonksiyona oldukça uzak olduğunu farkettiğimizde 2 fonksiyonunda özelliklerine sahip olduğunu hatırlayabiliriz.

slice(), splice()

Fonksiyonları hatırlayalım;

    
// Array elemanlarını dilimler.
['cat', 'dog', 'bird'].slice(2); // ['bird']
['cat', 'dog', 'bird'].slice(0, 2); // ['cat', 'dog']

// Array'e yeni eleman ekleyebilir veya değiştirebilir.
let pets = ['cat', 'dog'];
pets.splice(1, 0, 'bird'); // ["cat", "bird", "dog"]
pets.splice(1, 1, 'duck'); // ["cat", "duck", "dog"]

Aklımızda nasıl tutarız?

  • slice() fonksiyonunun son 4 harfini lime kelimesine benzetirsek array elemanlarını lime lime ettiğini hatırlayabiliriz :)
  • splice() fonksiyonunu hatırlamak için lime kelimesinden farklı olarak p harfine odaklanıp put kelimesi ile eşleştirebiliriz, bu sayede array’e ekleme yaptığını hatırlayabiliriz.

find(), filter()

Fonksiyonları hatırlayalım;

    
// Tek eşleşen eleman bulur.
[1,2,3].find(el => el > 1); // 2

// Tüm eşleşen elemanları bulur.
[1,2,3].filter(el => el > 1); // [2, 3]

Aklımızda nasıl tutarız?

Bu fonksiyonları hatırlamak için heceleme yöntemini deneyeceğiz. Eğer heceleme yöntemini kullanırsak tek heceli olan find() fonksiyonun tek eleman döndürdüğünü, iki heceli olan filter() fonksiyonun ise iki veya daha fazla eleman döndürdüğünü hatırlayabiliriz.
